## Git 運用ルール

### 基本方針

**コードを変更するたびに、必ず GitHub へプッシュすること。**

### 手順

1. 変更をステージング（個別ファイル指定を優先）
   ```
   git add <変更ファイル>
   ```
2. コミットメッセージは変更内容の「なぜ」を簡潔に記述
   ```
   git commit -m "メッセージ"
   ```
3. 即座にプッシュ
   ```
   git push
   ```

### コミットメッセージの規則

- 日本語 OK
- 1〜2文で「何を・なぜ変えたか」を書く
- 例：`ランディングページのCTAボタン文言を修正 — ユーザーテストのフィードバック反映`

### 禁止事項

- `--no-verify` でフックをスキップしない
- `--force` push は main/master ブランチへ行わない
- `.env` や認証情報ファイルをコミットしない
- まとめてコミットせず、変更のたびにこまめにプッシュする

---

## 作業ガイドライン

- 既存ファイルの編集を優先し、不要な新規ファイルを作らない
- コメントは「なぜそうしているか」が自明でない場合のみ記述
- セキュリティ上の懸念（XSS、インジェクション等）が生じた場合は即座に修正する
